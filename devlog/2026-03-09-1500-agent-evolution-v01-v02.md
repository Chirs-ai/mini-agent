# Mini-Agent 从构思到 v0.2.0 的演进

**时间**：2026-03-09 15:00
**版本**：v0.1.0 → v0.2.0

## 背景：从 pi 文章到自己动手

通过阅读两篇文章深入理解了 pi 编码智能体的设计哲学：

- [Mario Zechner — What I learned building an opinionated and minimal coding agent](https://mariozechner.at/posts/2025-11-30-pi-coding-agent/) (2025-11-30)
- [Armin Ronacher — Pi: The Minimal Agent Within OpenClaw](https://lucumr.pocoo.org/2026/1/31/pi/) (2026-01-31)
- [pi-mono/packages/agent 源码](https://github.com/badlogic/pi-mono/tree/main/packages/agent)

核心洞察：**一个有用的编码智能体不需要复杂基础设施**——只需一个循环、几个工具、零抽象。模型本身已经足够聪明，框架只需忠实地执行工具、传递结果。

## v0.1.0：极简 Agent 原型

### 设计

将 pi 的 TypeScript 架构（5 文件、双层消息系统、事件流）精炼为 Python 单文件 ~100 行：

```
messages = [user: task]
while True:
  ① LLM(messages) → response
  ② finished? → 结束
  ③ execute tools → results
  ④ append results → 回到 ①
```

### 关键决策

1. **OpenRouter 而非 Anthropic SDK** — 通过 OpenAI 兼容 API 调用任意模型，`.env` 切换
2. **LLM 接口独立封装** — `llm_openrouter.py` 返回标准化的 `ChatResult`/`ToolCall`，agent 不依赖特定 SDK
3. **Session Logger 低侵入** — 独立 `session_logger.py`，agent 只加 5 行胶水代码
4. **3 工具**：bash、write_file、read_file

### 工具格式适配

| | Anthropic 格式 | OpenAI 格式 |
|---|---|---|
| 工具定义 | `input_schema` | `function.parameters` |
| 响应解析 | `resp.content[].type == "tool_use"` | `msg.tool_calls[].function` |
| 结果回送 | `role: "user"` + tool_result 数组 | 每个独立 `role: "tool"` 消息 |
| system prompt | `system=` 参数 | `messages[0].role = "system"` |

`llm_openrouter.py` 的 `to_openai_tools()` 自动处理两种格式的转换。

### Collatz 猜想示例任务

选择"计算 Collatz 序列并可视化为 ASCII 山脉"作为 demo 任务，原因：
- 需要多步工具调用（计算 → 写脚本 → 运行）
- 不寻常，有趣
- 清晰展示 agent loop 的自我纠错能力

### Haiku vs Sonnet 对比实验

| | Haiku | Sonnet 4.6 |
|---|---|---|
| Turns | 5 | 18 |
| 可视化质量 | 错误（每序列独立画、数百行空白） | 正确（重叠山脉、重采样、填充） |
| 自我纠错 | 重复同一错误 2 次 | 逐步排查 Windows 环境差异 |
| 耗时 | ~11s | ~45s |

结论：**模型选择至关重要**，框架极简没问题，但模型能力决定产出质量。

## v0.2.0：edit 工具 + 任务输入方式

### 添加 edit 工具

实现与 pi 相同的 4 工具集：bash、write_file、read_file、**edit**。

```python
def execute_tool("edit", args):
    count = content.count(old_text)
    if count == 0: return "Error: not found"
    if count > 1: return "Error: matches N locations"
    content.replace(old_text, new_text, 1)
```

关键约束：`old_text` 必须在文件中**唯一匹配**，否则报错要求提供更多上下文。这防止了意外修改。

### 任务输入方式

从硬编码改为三种输入方式：

```python
if os.path.isfile(sys.argv[1]):   # 文件路径
    task = open(sys.argv[1]).read()
elif len(sys.argv) > 1:            # 内联参数
    task = " ".join(sys.argv[1:])
else:                              # 交互式
    task = input("Task: ")
```

预设任务放入 `tasks/` 目录：`collatz.txt`、`task_manager.txt`。

### Task Manager 实验（展示 edit 价值）

28 轮完成，agent 用 edit 工具 7 次精确修改现有文件：

1. `write_file` 创建 task_app.py 初始版本
2. `bash` 测试基本功能
3. `edit` × 7 逐步添加 priority + stats 功能
4. `write_file` 创建测试脚本
5. `bash` 运行 → 31/31 测试通过

edit 的 7 次调用：
1. 添加 `PRIORITY_ORDER` 常量
2. 扩展 `cmd_add` 读取 priority
3. 扩展 `cmd_list` 按优先级排序
4. 插入 `cmd_stats` 函数
5. argparse 添加 `--priority`
6. 添加 `stats` 子命令
7. dispatch dict 加入 `stats`

每次只动几行，不碰其他代码——精确编辑 vs 全量重写的优势一目了然。

### 修复的问题

| 问题 | 解决 |
|------|------|
| Windows stdout GBK 编码崩溃（emoji） | `sys.stdout.reconfigure(encoding="utf-8")` |
| 管道输入只读一行 | 改用文件路径参数 + `open().read()` |
| OpenRouter 返回畸形 JSON | `try/except json.JSONDecodeError` 容错 |
